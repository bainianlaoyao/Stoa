# Platform Priority

## Purpose

This document records which platforms beyond Reddit and X are suitable for Stoa outreach, and what motion fits each one.

## Priority Order

### 1. Discord

- fit: very high
- motion: join relevant Claude / Anthropic / AI coding communities and reply when session-management pain comes up
- why it fits: highest density of real users discussing live workflows, low friction, real-time conversations
- main risk: server-specific anti-promo rules must be checked first

### 2. Hacker News

- fit: very high
- motion: `Show HN` post when Stoa is ready, plus occasional highly relevant thread replies
- why it fits: strong technical audience, good match for a real engineering product
- main risk: one-shot launch dynamic; hype or weak framing gets punished hard

### 3. Lobsters

- fit: very high
- motion: submit a link post and participate deeply in comments
- why it fits: very strong audience alignment for terminal tooling and AI coding workflows
- main risk: invite-only, history required, self-promo must remain a minority of activity

### 4. Dev.to

- fit: high
- motion: write technical articles about the problem space, architecture, or lessons learned; mention Stoa naturally
- why it fits: article format is good for explaining the multi-agent/session-management problem in depth
- main risk: pure announcement-style posts are weak and may read like marketing

### 5. Product Hunt

- fit: medium-high
- motion: one-time launch post
- why it fits: explicitly designed for product launches and discovery
- main risk: better after presentation quality, README, screenshots, and onboarding are polished

### 6. GitHub

- fit: selective
- motion: use your own repo Discussions; submit to awesome lists; comment only where directly relevant such as feature-request issues
- why it fits: developers validate tools on GitHub first
- main risk: cross-project promotion in unrelated Issues/Discussions is spam

### 7. Chinese Communities

- fit: high
- motion:
  - `V2EX 分享创造`: one clear project post
  - `掘金`: technical article
  - `知乎`: answer relevant questions with substance
- why it fits: Chinese-language Claude Code / coding-agent content is still relatively thin, so good technical content stands out
- main risk: each community has its own posting-frequency and content-quality rules

## Not Recommended

### Stack Overflow

- reason: the strictest self-promo norms of the platforms reviewed
- why rejected: wrong product-discovery shape, hostile to tool promotion, very easy to trip spam boundaries even with disclosure

## Practical Sequence

If doing this manually, the best order is:

1. Discord
2. Hacker News
3. Dev.to
4. GitHub list/discussion opportunities
5. Chinese communities
6. Product Hunt when launch assets are strong
7. Lobsters when account history is ready

## Channel Shape

- fast feedback / high conversation density: Discord
- highest-quality technical launch: Hacker News
- long-form explanation: Dev.to, 掘金, 知乎
- direct builder discovery: Product Hunt
- highly technical niche audience: Lobsters
- project validation surface: GitHub
