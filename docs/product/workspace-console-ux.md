# 工作区控制台交互设计

## 产品定位

界面采用绝对二元分区：左侧为工作区控制台，右侧为主终端视图。这个布局不追求 IDE 式多面板复杂性，而是优先保证多个工作区之间的切换成本足够低。

## 左侧工作区控制台

左侧区域使用垂直卡片列表展示工作区。每张卡片至少包含：

- 工作区名称或路径别名。
- 最近一次回传的状态摘要。
- 运行状态灯。
- 当前是否为激活态。

卡片点击行为必须是全局唯一的工作区切换入口。后续即使增加快捷键，也只能作为附加能力，不能替代点击逻辑。

## 右侧主终端视图

右侧区域由 `xterm.js` 承载。终端实例不应在切换时被销毁，而应保持后台存活，仅通过视图显隐切换可见性，以确保切换零延迟且不丢缓冲。

## 状态反馈要求

- 空闲、运行中、错误、等待输入必须有可区分的视觉状态。
- 状态摘要来源必须是状态通道，不得从终端输出中猜测。
- UI 上的状态变化要与后端事件顺序一致，避免先展示成功、后收到失败的错序体验。

## 不做的事情

- 不引入复杂 command palette 作为主要交互入口。
- 不让用户在界面中直接编辑底层状态文件。
- 不在当前阶段设计多窗口或云同步能力。
