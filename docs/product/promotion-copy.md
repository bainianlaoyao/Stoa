- 近千个测试守住核心路径

我做了一个本地 AI 编程工作台 Stoa。

它解决一个很具体的问题：AI CLI 会话越来越多以后，项目、终端、状态和恢复路径会变得很乱。

Stoa 不替代 IDE，也不做云端 Agent 平台。它把 Claude Code、Codex、OpenCode 这类 CLI Agent 放进同一个本地桌面控制台里。

目前支持：

- 多项目 / 多工作区
- 多 CLI provider
- 后台会话保持
- 会话恢复
- 结构化状态侧信道
- 真实终端
- Electron E2E、生成式 Playwright journey、行为覆盖检查

项目还在快速演进阶段，欢迎真实使用 AI CLI 的朋友来挑毛病。
