# 核心交互流程

## 1. 创建工作区

1. 用户选择本地项目路径。
2. 系统为该路径创建工作区记录。
3. 主进程启动对应 PTY 与 CLI 进程。
4. 如果需要，注入 hook sidecar 到项目目录。
5. UI 新增工作区卡片并绑定终端视图。

## 2. 切换工作区

1. 用户点击左侧卡片。
2. 前端仅更新 `activeId`。
3. 对应终端视图显隐切换。
4. 后台工作区继续运行，不销毁 PTY。

## 3. 会话恢复

1. 应用启动时读取本地状态文件。
2. 主进程枚举工作区与上次记录的 `last_cli_session_id`。
3. 对每个工作区重新拉起 CLI，并将会话指针传给 CLI。
4. CLI 自行恢复内部历史。
5. UI 根据 `active_workspace_id` 恢复激活工作区，并根据最新状态事件恢复卡片状态与终端挂载关系。
6. 在 fresh event 到达前，启动态 UI 只视为 provisional state。

## 4. 状态事件更新

1. CLI 或外挂脚本捕获结构化事件。
2. 事件通过 HTTP POST 发往本地 Express webhook。
3. Session Manager 合成并落盘必要状态。
4. 主进程通过 IPC 向前端广播更新。
5. Pinia store 映射为工作区卡片和状态摘要。

## 5. 异常恢复

1. 某工作区 PTY 或 CLI 退出。
2. 主进程记录退出原因和时间戳。
3. UI 将工作区标记为错误或已停止。
4. 用户可触发重启，或等待统一恢复逻辑处理。
