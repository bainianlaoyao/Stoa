# 日志与调试策略

## 目标

调试体系要能回答三个问题：哪个工作区出了问题，问题发生在视觉流还是状态流，恢复流程卡在什么阶段。

## 日志分层

建议后续实现时至少分为：

- 应用启动日志
- PTY 生命周期日志
- Webhook 接收日志
- Session Manager 状态迁移日志
- 恢复流程日志

## 调试优先级

1. 先确认状态事件是否收到。
2. 再确认 Session Manager 是否合成了正确状态。
3. 最后再检查 UI 是否正确映射。

这个顺序符合 smart backend 原则，避免一上来就从前端表现倒推系统真相。

## 常见故障定位

- 终端有输出但状态灯不更新：优先检查 hook sidecar 与 webhook server。
- 状态灯更新但终端空白：优先检查 PTY 与 xterm.js 绑定。
- 重启后工作区消失：优先检查 `state.json` 是否被正确写入或路径是否失效。
