# 状态存储与恢复运维说明

## 默认存储位置

默认将最小恢复状态写入 `~/.vibecoding/state.json`。这个文件不是调试日志，而是恢复索引，因此必须保持结构稳定、体积可控。

## 运维原则

- 不把大体量终端历史直接塞入状态文件。
- 每次写入都尽量原子化，避免半写入状态。
- 状态版本字段必须保留，为将来迁移预留空间。

## 故障恢复建议

- 文件缺失：当作首次启动。
- 文件损坏：保留原文件，重命名为故障快照，再创建新文件。
- 工作区路径失效：跳过该工作区并提示。

## 备份建议

如果未来进入生产长期使用阶段，可增加轻量备份策略，例如每次关键写入前生成最近一次快照。但当前阶段不把多版本状态历史当作必须特性。
