# 风险登记册

## 高优先级风险

### 1. node-pty 原生依赖复杂度

影响：开发环境不一致时，工程初始化和打包都可能失败。

应对：尽早验证 native rebuild，保持 Electron 版本与本地工具链稳定。

### 2. 双通道模型被实现阶段破坏

影响：如果重新引入终端输出脏解析，系统状态将变得脆弱且不可预测。

应对：所有状态字段必须追溯到结构化事件来源。

### 3. Renderer 越权

影响：一旦渲染层直接控制文件系统或子进程，架构边界会快速失守。

应对：通过 preload 白名单 API 与代码审查双重限制。

### 4. 恢复逻辑过度设计

影响：如果过早追求完整缓冲恢复，可能拖慢整体交付并提高一致性风险。

应对：先坚持最小恢复模型，只保留工作区和 CLI session 指针。
