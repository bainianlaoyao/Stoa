# 构建与打包基线

## 目标

提前约束打包路径，避免实现阶段只关注开发模式而忽略桌面应用最终形态。

## 基线要求

- 支持开发模式本地启动。
- 支持生产模式构建 renderer 静态资源。
- 支持 Electron 主进程打包。
- 支持 `node-pty` 原生模块在目标 Electron 版本下正确加载。

## node-pty 特殊处理

`node-pty` 是本项目打包复杂度最高的依赖之一。后续实现时必须验证：

- native addon 已按 Electron ABI rebuild。
- 打包后 native 产物未被错误封入不可执行环境。
- Windows 环境下 PTY 相关辅助文件能正常访问。

## 验证清单

进入实现阶段后，每次构建链路调整都应至少验证：

1. 应用能启动。
2. 能创建一个工作区。
3. PTY 能接收输入并输出到 `xterm.js`。
4. Hook 事件能经 Express 回传并驱动 UI 更新。
5. 重启应用后能读取状态文件并尝试恢复 session。
