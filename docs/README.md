# Vibecoding 极简多会话管理框架文档库

本仓库用于沉淀 Vibecoding 极简多会话管理框架的设计资产，而不是直接承载应用代码。当前阶段的目标是先把产品哲学、系统架构、环境基线、运行约束和执行计划固定下来，让后续实现始终围绕同一套工程契约推进。

## 文档导航

- `overview/`：项目愿景、设计原则、术语表与路线图。
- `product/`：产品形态、交互模型和关键用户流程。
- `architecture/`：双通道模型、生命周期恢复、模块边界、扩展机制等核心架构设计。
- `architecture/` 中同时包含事件契约、状态机、provider 能力边界等实现前必须钉死的系统契约。
- `engineering/`：技术选型依据、本地开发环境、仓库结构和工程约定。
- `operations/`：状态持久化、日志调试、风险与恢复策略。
- `plans/`：面向后续实现阶段的里程碑与执行计划。

## 当前阶段范围

当前仓库是一个完全空白的 greenfield 项目，因此本阶段聚焦于三件事：

1. 把白皮书里的设计哲学转成正式文档。
2. 把未来代码仓库的结构与边界提前定义清楚。
3. 给出可以直接启动 Electron + Vue + node-pty 工程的环境基线。

## 不在当前阶段内的事项

- 不编写应用运行时代码。
- 不初始化 Electron/Vite 源码脚手架。
- 不加入与白皮书无关的功能扩展。

下一步应基于 `docs/plans/bootstrap-plan.md` 进入工程初始化与分阶段实现。
