实现结果是：
`App.vue` 不再硬编码依赖 Electron preload 来源，只依赖 `RendererApi` 形状的 `window.stoa` bridge，所以 web 模式下现有 `StoaClientPreloadAdapter` 也能完成 bootstrap、session graph 订阅、update/memory/title notifications、observability hydrate。`main.ts` 现在只在 `isStoaClientMode()` 下执行 web bootstrap，并注册 Pinia 的 `stoaClientPlugin()`。`git.ts`、`search.ts`、`update.ts`、`useFileTree.ts`、`useFileOperations.ts` 都改成了和 `settings.ts` / `workspaces.ts` 一致的 dual-path，优先 `isStoaClientMode()+getStoaClient()`，否则走 legacy bridge。`update.ts` 在 web 下仍沿用现有 adapter/stub 行为，没有去假定 server 已经有 update 路由。
